# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repo is

A community-maintained, **unofficial** reference for the Skylight app's HTTP API, reverse-engineered from observed traffic. Two artifacts:

1. **The OpenAPI spec** (`docs/openapi/openapi.yaml`, OpenAPI 3.0.3) — the source of truth that everything else derives from.
2. **A generated Go CLI** (`cli/`) — produced from the spec by [`onlycli`](https://github.com/onlycli/onlycli).

The spec documents only what has been seen in real captures. `TODO.md` tracks known-missing endpoints; each gap requires a HAR capture or other evidence before it can be filled in — do not invent endpoints, params, or schemas.

## Repo layout

- `docs/openapi/openapi.yaml` — the spec. **Primary thing you edit.** The HTML viewers in `docs/` load it via the relative path `openapi/openapi.yaml`.
- `examples/` — redacted request/response pairs.
- `captures/` — raw `.har` traffic captures used as evidence.
- `docs/` — guides (`auth.md`), static spec viewers (`swagger.html`, `redoc.html`).
- `cli/` — **fully generated**; see below.
- `.github/workflows/` — GitHub issue/project automation (daily-brief, triage, housekeeping, etc.); unrelated to the API itself.
- Version control is **jj** (Jujutsu), not git — there is a `.jj/` directory and no `.git/`.

## Tooling

Tools are pinned via `mise.toml` (`mise install` to get them):
- `onlycli` — generates the CLI from the spec.
- `vacuum` — fast OpenAPI linter.

## Common commands

Lint the spec (do this after editing the spec):
```bash
vacuum lint docs/openapi/openapi.yaml
```

Regenerate the CLI from the spec (do this after editing the spec — never hand-edit generated code):
```bash
onlycli generate --spec docs/openapi/openapi.yaml --name skylight --out cli
env -C cli go mod tidy
```
or
```bash
mise run generate
```

Build / run the CLI:
```bash
go build -o skylight cli && ./skylight --help
```
or
```bash
mise run build
```

Preview the docs site locally:
```bash
python3 -m http.server 8080   # then open http://localhost:8080/docs/swagger.html
```

## CLI architecture

Everything under `cli/commands/` and `cli/runtime/` is marked `// Code generated by onlycli; DO NOT EDIT.`. To change CLI behavior, edit the spec and regenerate — do not patch the Go files.

- **Command tree** (cobra): one group per OpenAPI tag (`chores`, `lists`, `calendars`, …). Each operation becomes a subcommand named after its method+path, e.g. `chores get-api-frames-frame-id`. Path params (`{frameId}`) become required string flags (`--frame-id`); query params become optional flags. `cli/commands/register.go` wires the whole tree; per-operation files are named `<tag>_<method>-<path>.go`.
- **Runtime** (`cli/runtime/`): `client.go` (HTTP, retries, pagination, streaming, output formatting via the global `--format`/`--transform`/`--template` flags), `auth.go` (OAuth2 device-code and client-credentials flows), `config.go`, `output.go`.
- **Config**: profile-based, stored at `~/.config/skylight/config.json` (mode 0600). Keys: `token`, `base_url`, `auth_type`. Managed via `skylight config set/get/list/use-profile` and `skylight auth login/status/logout`. Resolution order for base URL/token: profile → `SKYLIGHT_BASE_URL`/`SKYLIGHT_TOKEN` env → default `https://app.ourskylight.com`.

## API conventions (from observed traffic)

- Most resources are nested under `/api/frames/{frameId}/...` and follow **JSON:API** (`type`, `id`, `attributes`, `relationships`).
- Auth is an opaque OAuth 2.0 Bearer token (not a JWT); access tokens expire after 7200s and refresh tokens rotate on every use. Some older traffic uses `Authorization: Basic <opaque token>` (not username:password). See `docs/auth.md`.

## Editing rules

- Keep responses aligned with JSON:API. Use OpenAPI 3.1-style/3.0.3 idioms already present in the spec; put observed enums on params.
- Document only observed behavior; phrase descriptions objectively ("observed ...").
- **Redaction is mandatory**: never commit real tokens, emails, phone numbers, addresses, GPS, or IDs tied to real people. Replace with `REDACTED` (or stable placeholders like `CATEGORY_REDACTED`) while preserving structure. `examples/*-raw*.json` is git-ignored for un-redacted scratch files.
- Prefer small, focused changes (one endpoint at a time).
